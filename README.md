# DDX Зал — учёт весов

PWA для записи подходов в зале DDX Fitness: сканируешь QR на тренажёре → выбираешь упражнение → вводишь вес и повторы → данные хранятся локально (офлайн). Экспорт и резервная копия — в файл.

Ставится на домашний экран iPhone как обычное приложение. App Store и аккаунт разработчика не нужны.

## Возможности

- **Упражнения** — первоклассная сущность. На один QR можно завести несколько упражнений (многозадачный тренажёр) — после скана выбираешь нужное. Можно создавать упражнения и без QR (гантели, свободные веса).
- **Единицы кг/lbs** — переключатель на вводе; всё конвертируется и хранится в кг.
- **Множитель веса ×1/×2** — для тренажёров, где вес навешивается на каждую руку.
- **Стрелки ▲/▼** у полей веса и повторов — менять значение кнопками, не вводом.
- **Управление тренажёрами** — переименовать, привязать/сменить QR, удалить.
- **Статистика и график** прогресса веса по датам для каждого упражнения.
- **Журнал** сгруппирован по упражнениям внутри каждого дня.
- **Экспорт** в Excel/CSV и **резервная копия .json** с обратным импортом.

## Стек

- React + Vite + TypeScript
- `vite-plugin-pwa` — манифест, service worker, офлайн, установка
- `@zxing/browser` — сканер QR через камеру
- `Dexie` (IndexedDB) — локальная база
- `xlsx` (SheetJS) — экспорт/импорт Excel/CSV

## Запуск

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # сборка в dist/
npm run preview    # предпросмотр собранной версии
```

> Камера (`getUserMedia`) работает только в защищённом контексте: `localhost` или **HTTPS**. На самом iPhone тестировать удобнее всего после деплоя на HTTPS-хостинг или через https-туннель.

## Деплой (Netlify)

Подключён через git-репозиторий. Настройки сборки в [`netlify.toml`](netlify.toml): команда `npm run build`, каталог `dist`, SPA-редирект на `index.html`. Переменные окружения не нужны — приложение работает полностью локально.

## Установка на iPhone

1. Открой адрес деплоя в **Safari** на iPhone.
2. «Поделиться» → «На экран „Домой"».
3. Запусти с домашнего экрана — приложение откроется на весь экран, камера и офлайн работают (iOS 16.4+).

## Данные: экспорт и импорт

На вкладке «Данные»:

- **Excel (.xlsx) / CSV** — выгрузка всех подходов (в т.ч. колонка `ISO` для точного реимпорта).
- **Резервная копия (.json)** — полный дамп (упражнения + сессии + подходы), точный round-trip.
- **Импорт** — загрузка `.json` (восстановление, слияние по `id`) либо `.xlsx`/`.csv` (упражнения переиспользуются по имени, дубли подходов пропускаются).

## Структура

```
src/
  data/catalog.json       каталог тренажёров (Фаза 0)
  lib/
    catalog.ts            разбор QR → номер → тренажёр, codeKey
    units.ts              кг ↔ lbs, шаги степпера
    stats.ts              агрегаты по дням (макс/тоннаж/1ПМ)
    export.ts             выгрузка в Excel/CSV
    backup.ts             JSON-бэкап + импорт (json/xlsx/csv)
    id.ts, format.ts      утилиты
  components/
    Scanner.tsx           сканер камеры
    controls.tsx          Stepper, SegmentedControl
    ProgressChart.tsx     inline-SVG график прогресса
    ExerciseForm.tsx      создание/редактирование упражнения
    ScanChooser.tsx       выбор упражнения при нескольких на одном коде
  screens/
    LogScreen.tsx         запись подходов
    HistoryScreen.tsx     журнал (день → упражнения)
    ManageScreen.tsx      управление + статистика/график
    DataScreen.tsx        экспорт и импорт
  db.ts                   Dexie (exercises, sets, sessions)
  types.ts                типы предметной области
  App.tsx                 оболочка + навигация + разбор скана
```
